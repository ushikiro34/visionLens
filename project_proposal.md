# K-Food Vision Lens 제안서
> 버전: 1.1 (2026년 3월 · Human-in-the-Loop 기본 룰 버전)

---

## 1. 프로젝트 개요

**프로젝트명**
K-Food Vision Lens — 데이터 신뢰성 우선 버전

**핵심 목표**
사용자가 직접 촬영한 한 끼 식사 사진을 통해
한국 음식문화(뚝배기, 식판, 반찬 복합 등)에 최적화된 실제 섭취 칼로리·영양 정보를 제공하되,
**모든 결과의 근거가 공공기관 공식 데이터에만 기반**하며,
**Human-in-the-Loop**을 절대적 기본 룰로 삼아 AI의 불확실성을 사용자 판단으로 보정한다.

**절대 타협 불가 기본 룰 (Non-negotiable Principle)**

Human-in-the-Loop (HITL)
- AI가 추정한 모든 값(Fill Ratio, 음식 종류, 그릇 종류, 칼로리 등)에 대해
  사용자가 **반드시 확인·수정**할 수 있는 인터페이스를 항상 제공
- AI의 신뢰도 점수가 일정 수준 이하일 경우 **자동으로 사용자 보정 모드** 진입
- 최종 섭취 기록은 **사용자가 최종 승인한 값**만 저장·동산 됨
  (AI 자동 결정값은 기록되지 않음)

---

## 2. 프로젝트 가치 및 우선순위

1. 데이터 신뢰성 (최우선)
2. Human-in-the-Loop 절대 원칙
3. 한국 음식문화 특화 정밀도 (뚝배기/식판/복합 반찬)
4. 법령·규정 준수 용이성
5. 비용 효율성 (Gemini 등 외부 API 최소화)

---

## 3. 주요 차별화 포인트

- 모든 영양 정보 = 식약처 공식 DB만 사용 (다른 출처 절대 배제)
- 모든 학습 데이터 = AI Hub 공공 한국 음식 이미지 데이터셋만 사용
- AI hallucination 방지 → 오픈소스 YOLO + 공식 매핑 + 사용자 검증
- 결과마다 **출처·버전·신뢰도 점수·Human 수정 이력** 기록 및 표시
- 사용자 보정값을 다음 모델 fine-tuning의 ground truth로 재활용 (선택적)

---

## 4. 기대 효과

- 민원·신뢰도 라벨 최소화
- 당뇨·다이어트·영양 관리 사용자에게 신뢰할 수 있는 참고 자료 제공
- 식약처·보건복지부 등 규제 기관 심사 시 유리한 근거 자료 확보
- 글로벌 앱 시장 내 한국식 그릇·식판·국물 요리 인식률 선도

---

## 5. 성공 지표 (KPI)

| 지표 | 목표 |
|------|------|
| HITL 보정률 평균 | 35% 이하 (너무 높으면 AI 성능 문제) |
| 사용자 최종 승인 후 칼로리 오차율 | ±10% 이내 (자체 검증 기준) |
| 데이터 출처 신뢰도 점수 평균 | 95점 이상 |
| 식약처 DB 동기화 지연 | 0일 (분기 자동 업데이트) |

---

## 6. 리스크 및 대응

| 리스크 | 대응 방식 |
|--------|---------|
| AI가 한국 음식을 잘못 인식 | Human-in-the-Loop 강제 + Smart Guide 제공 |
| 데이터 신뢰성 라벨 | 모든 결과에 식약처 출처·버전·링크 자동 표시 |
| 사용자 보정 귀찮음 → 이탈 | 직관적 Quantity Slider + AR Overlay 최적화 |
| 모델 성능 저하 | 사용자 보정 데이터를 주기적으로 fine-tune에 반영 |
| 오프라인 환경 | 로컬 캐시 + 앱 번들 경량 DB + HITL 직접 입력 폴백 |
